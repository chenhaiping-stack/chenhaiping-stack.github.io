在GitHub Pages上部署静态博客

2023年10月15日 作者: 张三 分类: 前端开发

GitHub Pages 是一个静态站点托管服务，直接从 GitHub 上的仓库获取 HTML、CSS 和 JavaScript 文件，然后发布网站。您可以在 GitHub Pages 示例集合中查看 GitHub Pages 站点的示例。

本教程将指导您如何创建一个静态博客页面，并通过 GitHub Pages 进行部署。我们将重点介绍在 HTML 页面中使用绝对路径和相对路径引用图片的方法。

路径类型说明

绝对路径：使用完整的 URL 地址引用资源（例如：https://example.com/images/pic.jpg）

相对路径：根据当前文件位置引用资源（例如：../images/pic.jpg）

图片路径示例

HTML/CSS 开发

此图片使用绝对路径引用，通过完整的 URL 地址加载网络图片资源。

绝对路径
<img src="https://example.com/image.jpg">

编程概念

此图片使用相对路径引用，图片文件位于本地目录结构中。

相对路径
<img src="../images/local-image.jpg">

部署到 GitHub Pages 的步骤

  1. 在 GitHub 上创建一个新仓库，命名为 username.github.io（username 是你的 GitHub 用户名）
  2. 将你的 HTML、CSS、JavaScript 和图片文件添加到仓库中
  3. 在仓库设置中，找到 GitHub Pages 部分，选择要部署的分支（通常是 main 或 master）
  4. 保存设置后，你的网站将在几分钟内上线，地址为 https://username.github.io
  5. 对于自定义域名，可以在设置中添加 CNAME 文件

路径使用注意事项

在 GitHub Pages 项目中，使用相对路径时需要注意文件结构：

  • 图片文件应放在项目目录中（如 /images 文件夹）
  • HTML 文件中引用图片时，路径应相对于 HTML 文件的位置
  • 绝对路径适用于引用外部资源或 CDN 上的文件
  • 相对路径更适用于项目内部的资源引用
